# Pokémon API Automation Project

A complete, production-grade shell scripting pipeline that demonstrates advanced Bash techniques by interacting with the **PokéAPI** (<https://pokeapi.co>).

This project covers the full data-engineering lifecycle:
- Single & batch API fetching
- JSON parsing with `jq`, `awk`, `sed`
- Retry logic & robust error handling
- Parallel processing with background jobs
- Reporting & CSV generation

Perfect for DevOps, Data Engineering, and Systems Programming learning paths.

## Project Structure

```
.
├── apiAutomation-0x00                  # Task 0 – Fetch single Pokémon (Pikachu)
├── data.json                           # Raw response from Task 0
├── errors.txt                          # Execution log (success/failure timestamps)
├── data_extraction_automation-0x01     # Task 1 – Parse name, type, height, weight using only jq|awk|sed
├── batchProcessing-0x02                # Task 2 – Sequential batch fetch with 3× retry logic
├── batchProcessing-0x04                # Task 4 – Parallel fetch with concurrency control & PID cleanup
├── summaryData-0x03                    # Task 3 – Generate CSV report + average height/weight
├── pokemon_data/                       # Directory containing all fetched JSON files
│   ├── bulbasaur.json
│   ├── ivysaur.json
│   ├── venusaur.json
│   ├── charmander.json
│   └── charmeleon.json
└── pokemon_report.csv                  # CSV summary generated by summaryData-0x03
```

## Scripts Overview

| Script                            | Purpose                                                                 | Key Features                                      |
|-----------------------------------|-------------------------------------------------------------------------|---------------------------------------------------|
| `apiAutomation-0x00`              | Fetch Pikachu data                                                      | Strict mode, logging, no `jq` dependency          |
| `data_extraction_automation-0x01` | Extract & format name, type, height, weight                             | Uses **only** `jq`, `awk`, `sed` (as required)    |
| `batchProcessing-0x02`            | Sequential fetch of 5 Pokémon with **3 retries**                        | `MAX_RETRIES=3`, failure logging, cleanup        |
| `batchProcessing-0x04`            | **Parallel** fetch using background jobs (`&`) + `wait`                | Concurrency limit, PID tracking, `wait -n`        |
| `summaryData-0x03`                | Generate `pokemon_report.csv` + calculate averages                      | Proper capitalization, unit conversion, clean CSV |

## How to Run (step by step)

```bash
# 1. Fetch Pikachu (Task 0)
./apiAutomation-0x00

# 2. Extract and display human-readable info (Task 1)
./data_extraction_automation-0x01
# → Pikachu is of type Electric, weighs 6kg, and is 0.4m tall.

# 3. Batch download 5 Pokémon with retry logic (Task 2)
./batchProcessing-0x02

# 4. Generate summary report (Task 3)
./summaryData-0x03
# → Creates pokemon_report.csv + prints averages

# 5. Ultra-fast parallel download (Task 4)
./batchProcessing-0x04
```
